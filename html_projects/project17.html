HTML Entities and Special Characters

HTML entities are special codes used to display reserved characters and symbols that might otherwise be interpreted as HTML code. This page demonstrates various HTML entities and special characters.

Why Use HTML Entities?

There are three main reasons to use HTML entities:

  1. Reserved Characters: Some characters are reserved in HTML and must be replaced with entities to display correctly.
  2. Invisible Characters: Some characters are not visible or might not be available on the keyboard.
  3. Special Symbols: Many symbols don't have dedicated keys on standard keyboards.

Reserved Characters

These characters have special meaning in HTML and must be represented as entities to display them as text:

Character Entity Name Entity Number Description
< &lt; &#60; Less than sign
> &gt; &#62; Greater than sign
& &amp; &#38; Ampersand
" &quot; &#34; Double quotation mark
' &apos; &#39; Single quotation mark (apostrophe)

Example Usage

To display HTML code as text, you need to use entities:

<p>This is a paragraph.</p>

The above is displayed using entities like this:

&lt;p&gt;This is a paragraph.&lt;/p&gt;

Entity Formats

HTML entities can be represented in several formats:

Named Entities

Named entities start with an ampersand (&), followed by a name, and end with a semicolon (;).

Example: &copy; displays as ©

Numeric Entities (Decimal)

Decimal entities start with an ampersand (&), followed by a hash (#), then a decimal number, and end with a semicolon (;).

Example: &#169; displays as ©

Numeric Entities (Hexadecimal)

Hexadecimal entities start with an ampersand (&), followed by a hash and an x (#x), then a hexadecimal number, and end with a semicolon (;).

Example: &#xA9; displays as ©

Which Format to Use?

  • Named entities are easier to remember but are not available for all characters.
  • Decimal entities are widely supported and work in all browsers.
  • Hexadecimal entities are useful when working with character codes from specifications.
