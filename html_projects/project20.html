HTML Video

HTML5 introduced the <video> element, which allows you to embed video content in your web pages without requiring plugins like Flash. This page demonstrates how to use the video element and its various attributes.

Basic Video Element

The basic syntax for the video element is:

<video src="video-file.mp4" controls></video>

Here's a simple example:

The code for this video player is:

<video src="https://www.w3schools.com/html/mov_bbb.mp4" controls width="400">
    Your browser does not support the video element.
</video>

Video Attributes

The video element supports several attributes that control its behavior:

controls

The controls attribute adds video controls, like play, pause, volume, and fullscreen.

autoplay

The autoplay attribute makes the video start playing as soon as it's ready. Note that many browsers block autoplay unless the user has interacted with the page or the video is muted.

Note: This example includes muted to allow autoplay in most browsers.

loop

The loop attribute makes the video play again every time it finishes.

muted

The muted attribute specifies that the video output should be muted by default.

preload

The preload attribute specifies if and how the video should be loaded when the page loads.

  • auto - Load the entire video file
  • metadata - Load only metadata
  • none - Don't preload anything

width and height

These attributes specify the dimensions of the video player in pixels.

playsinline

The playsinline attribute is especially important for iOS devices, as it allows the video to play inline rather than automatically entering fullscreen mode.

Multiple Sources

You can specify multiple video sources using the <source> element inside the video element. The browser will use the first recognized format.

The code for this video player is:

<video controls width="400">
    <source src="mov_bbb.mp4" type="video/mp4">
    <source src="mov_bbb.ogg" type="video/ogg">
    Your browser does not support the video element.
</video>

Video Formats

Different browsers support different video formats. The most common formats are:

Format MIME Type Browser Support
MP4 (H.264) video/mp4 All modern browsers
WebM video/webm Chrome, Firefox, Opera
OGG video/ogg Chrome, Firefox, Opera

To ensure maximum compatibility, it's best to provide at least MP4 and WebM formats.

Best Practices

  • Always provide controls unless there's a specific reason not to.
  • Avoid autoplay for videos with sound, as it can be annoying and may be blocked by browsers.
  • Provide multiple formats for maximum browser compatibility.
  • Include fallback content for browsers that don't support the video element.
  • Specify the type attribute in source elements to help browsers choose the right format.
  • Add a poster image to give users a preview before they play the video.
  • Make videos responsive to ensure they work well on all devices.
  • Consider preloading options to balance performance and data usage.
  • Optimize video files for web use to reduce file size.
  • Add the playsinline attribute for better behavior on mobile devices.
  • Include captions or subtitles for accessibility.

Accessibility Considerations

  • Provide captions or subtitles using the <track> element
  • Ensure keyboard accessibility for all video controls
  • Use ARIA attributes when creating custom video controls
  • Provide transcripts for important video content
  • Consider users who may have video disabled or unavailable
