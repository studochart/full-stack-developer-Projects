HTML Audio

HTML5 introduced the <audio> element, which allows you to embed audio content in your web pages without requiring plugins like Flash. This page demonstrates how to use the audio element and its various attributes.

Audio Attributes

The audio element supports several attributes that control its behavior:

controls

The controls attribute adds audio controls, like play, pause, and volume.

autoplay

The autoplay attribute makes the audio start playing as soon as it's ready. Note that many browsers block autoplay unless the user has interacted with the page or the audio is muted.

Note: This example includes muted to allow autoplay in most browsers.

loop

The loop attribute makes the audio play again every time it finishes.

muted

The muted attribute specifies that the audio output should be muted by default.

preload

The preload attribute specifies if and how the audio should be loaded when the page loads.

  • auto - Load the entire audio file
  • metadata - Load only metadata
  • none - Don't preload anything

Multiple Sources

You can specify multiple audio sources using the <source> element inside the audio element. The browser will use the first recognized format.

The code for this audio player is:

<audio controls>
    <source src="horse.ogg" type="audio/ogg">
    <source src="horse.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

Audio Formats

Different browsers support different audio formats. The most common formats are:

Format MIME Type Browser Support
MP3 audio/mpeg All modern browsers
OGG audio/ogg Chrome, Firefox, Opera
WAV audio/wav Chrome, Firefox, Safari, Opera
AAC audio/aac Chrome, Safari

To ensure maximum compatibility, it's best to provide at least MP3 and OGG formats.

Best Practices

  • Always provide controls unless there's a specific reason not to (like background music with a separate control interface).
  • Avoid autoplay for audio with sound, as it can be annoying and may be blocked by browsers.
  • Provide multiple formats for maximum browser compatibility.
  • Include fallback content for browsers that don't support the audio element.
  • Specify the type attribute in source elements to help browsers choose the right format.
  • Consider preloading options to balance performance and data usage.
  • Optimize audio files for web use to reduce file size.
  • Provide accessible controls for users who rely on keyboard navigation or screen readers.

Accessibility Considerations

  • Provide transcripts for important audio content
  • Use ARIA attributes when creating custom audio controls
  • Ensure keyboard accessibility for all audio controls
  • Consider users who may have audio disabled or unavailable
