Introduction to Semantic HTML

Semantic HTML elements clearly describe their meaning to both the browser and the developer. They provide information about the content they contain, rather than just how to display it.

HTML5 introduced many semantic elements to help structure web pages in a more meaningful way, making them more accessible and easier to maintain.

Semantic vs. Non-Semantic Elements

Non-Semantic Elements

Non-semantic elements don't tell anything about their content. Examples include:

  • <div> - Defines a division or section
  • <span> - Defines an inline section

Example of non-semantic markup:

<div class="header">
    <div class="title">My Website</div>
    <div class="nav">
        <div class="nav-item">Home</div>
        <div class="nav-item">About</div>
    </div>
</div>
<div class="main-content">
    <div class="article">
        <div class="article-title">Article Title</div>
        <div class="article-content">Content goes here...</div>
    </div>
</div>
<div class="footer">Copyright 2023</div>

Semantic Elements

Semantic elements clearly define their content. Examples include:

  • <header> - Defines a header section
  • <nav> - Defines navigation links
  • <main> - Defines the main content
  • <article> - Defines an independent, self-contained content
  • <section> - Defines a section of content
  • <footer> - Defines a footer section

Example of semantic markup:

<header>
    <h1>My Website</h1>
    <nav>
        <ul>
            <li><a href="#">Home</a></li>
            <li><a href="#">About</a></li>
        </ul>
    </nav>
</header>
<main>
    <article>
        <h2>Article Title</h2>
        <p>Content goes here...</p>
    </article>
</main>
<footer>Copyright 2023</footer>

Common Semantic Elements

<header>

The <header> element represents introductory content, typically a group of introductory or navigational aids. It may contain heading elements, a logo, a search form, an author name, and other elements.

This is a header

Contains introductory content for a section or page

<nav>

The <nav> element represents a section of a page that contains navigation links, either within the current document or to other documents.

  • Home
  • About
  • Services
  • Contact

<main>

The <main> element represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document.

There should be only one <main> element in a document, and it should not be a descendant of <article>, <aside>, <footer>, <header>, or <nav>.

<article>

The <article> element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable.

Article Title

This is an article content. It represents a self-contained piece of content that could be distributed independently.

Examples include: a forum post, a magazine or newspaper article, a blog entry, a user-submitted comment, etc.

<section>

The <section> element represents a standalone section of content that doesn't have a more specific semantic element to represent it.

Section Title

This is a section of content. Sections usually have a heading.

A section could be a chapter, a tabbed content area, or a numbered section of a thesis.

<aside>

The <aside> element represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.

This is the main content area where the primary content would go. The aside element would typically appear alongside this content.

Related Info

This is an aside with content tangentially related to the main content.

<footer>

The <footer> element represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of the section, copyright data, or links to related documents.

© 2023 Example Company. All rights reserved.

Contact: info@example.com

Other Semantic Elements

  • <figure> and <figcaption> - For images, diagrams, etc., with a caption
  • <time> - For dates and times
  • <mark> - For highlighted text
  • <details> and <summary> - For additional details that can be shown or hidden
  • <address> - For contact information

Page Structure Example

Here's how semantic elements can be used to structure a complete web page:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Semantic HTML Example</title>
</head>
<body>
    <header>
        <h1>Website Title</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
                <li><a href="#">Services</a></li>
                <li><a href="#">Contact</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <section>
            <h2>Welcome Section</h2>
            <p>Introduction to the website.</p>
        </section>

        <section>
            <h2>Latest Articles</h2>
            
            <article>
                <h3>Article Title 1</h3>
                <p>Article content goes here...</p>
                <time datetime="2023-07-15">July 15, 2023</time>
            </article>
            
            <article>
                <h3>Article Title 2</h3>
                <p>Article content goes here...</p>
                <time datetime="2023-07-10">July 10, 2023</time>
            </article>
        </section>

        <aside>
            <h2>Related Information</h2>
            <ul>
                <li>Related Link 1</li>
                <li>Related Link 2</li>
                <li>Related Link 3</li>
            </ul>
        </aside>
    </main>

    <footer>
        <p>© 2023 Example Website. All rights reserved.</p>
        <address>
            Contact: <a href="mailto:info@example.com">info@example.com</a>
        </address>
    </footer>
</body>
</html>

Benefits of Semantic HTML

  • Accessibility - Screen readers and other assistive technologies can better understand the content structure.
  • SEO - Search engines give more weight to important content when it's properly marked up.
  • Maintainability - Code is easier to read and maintain when it clearly describes its purpose.
  • Reusability - Semantic elements make it easier to identify and reuse specific parts of the page.
  • Mobile Friendliness - Many semantic elements have built-in responsive behavior.
  • Future Compatibility - Semantic markup is more likely to remain compatible with future web technologies.

Best Practices

  • Use semantic elements whenever possible instead of generic div or span elements.
  • Use heading elements (h1 through h6) to create a logical document outline.
  • Use article for content that can stand alone, and section for grouping related content.
  • Use nav for major navigation blocks, not for all groups of links.
  • Use header and footer within sections or articles when appropriate, not just for the page.
  • Use main only once per page to identify the primary content area.
  • Use aside for content that is tangentially related to the main content.
  • Use figure and figcaption for images, diagrams, or code snippets that are referenced in the main content.
