HTML Meta Tags

Meta tags provide metadata about an HTML document. They don't appear on the page itself but are used by browsers, search engines, and other web services. This page explains the most important meta tags and how to use them.

Basic Meta Tags

These are the fundamental meta tags that should be included in most HTML documents:

Character Encoding

<meta charset="UTF-8">

This specifies the character encoding for the HTML document. UTF-8 is the recommended encoding as it can handle any character.

Viewport

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This controls how the page is displayed on mobile devices. It's essential for responsive web design.

Description

<meta name="description" content="A brief description of the page content.">

This provides a short description of the page, which may be used by search engines in search results.

Author

<meta name="author" content="Author Name">

This specifies the name of the page's author.

SEO Meta Tags

These meta tags can help improve your page's search engine optimization (SEO):

Keywords

<meta name="keywords" content="keyword1, keyword2, keyword3">

This lists keywords relevant to the page content. Note that many search engines, including Google, no longer give this much weight.

Robots

<meta name="robots" content="index, follow">

This tells search engines whether to index the page and follow its links. Options include:

  • index - Allow the page to be indexed
  • noindex - Prevent the page from being indexed
  • follow - Follow links on the page
  • nofollow - Don't follow links on the page
  • noarchive - Don't show a cached link in search results
  • nosnippet - Don't show a snippet in search results

Canonical URL

<link rel="canonical" href="https://example.com/original-page">

This isn't a meta tag but serves a similar purpose. It tells search engines which URL is the "canonical" or preferred version of a page, which helps prevent duplicate content issues.

Viewport Meta Tag

The viewport meta tag is crucial for responsive web design. Let's explore it in more detail:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

The content attribute can include several values:

  • width=device-width - Sets the width of the viewport to the width of the device
  • initial-scale=1.0 - Sets the initial zoom level when the page is first loaded
  • minimum-scale=1.0 - Sets the minimum zoom level
  • maximum-scale=1.0 - Sets the maximum zoom level
  • user-scalable=no - Prevents the user from zooming in or out (not recommended for accessibility reasons)

Example with additional values:

<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=2.0">

Social Media Meta Tags

These meta tags control how your page appears when shared on social media platforms:

Open Graph (Facebook, LinkedIn, etc.)

<meta property="og:title" content="Page Title">
<meta property="og:description" content="Page description">
<meta property="og:image" content="https://example.com/image.jpg">
<meta property="og:url" content="https://example.com/page">
<meta property="og:type" content="website">

Open Graph meta tags control how your page appears when shared on Facebook and many other platforms. The og:image should be at least 1200×630 pixels for best results.

Twitter Cards

<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Page Title">
<meta name="twitter:description" content="Page description">
<meta name="twitter:image" content="https://example.com/image.jpg">

Twitter Card meta tags control how your page appears when shared on Twitter. The twitter:card can be summary, summary_large_image, app, or player.

HTTP-Equiv Meta Tags

These meta tags provide HTTP header information:

Refresh

<meta http-equiv="refresh" content="5;url=https://example.com">

This redirects the page to a new URL after a specified number of seconds (5 in this example).

X-UA-Compatible

<meta http-equiv="X-UA-Compatible" content="IE=edge">

This tells Internet Explorer to use the latest rendering engine. It's less important now but was crucial for older versions of IE.

Content-Type

<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">

This specifies the document's MIME type and character encoding. The <meta charset="UTF-8"> tag is the modern, shorter equivalent.

Content Security Policy

<meta http-equiv="Content-Security-Policy" content="default-src 'self'">

This helps prevent cross-site scripting (XSS) attacks by specifying which content sources are allowed.

Other Meta Tags

Theme Color

<meta name="theme-color" content="#4285f4">

This sets the color of the browser UI when the page is displayed as a Progressive Web App or in mobile browsers that support it.

App Links

<meta property="al:ios:app_name" content="App Name">
<meta property="al:ios:app_store_id" content="123456789">
<meta property="al:android:package" content="com.example.app">

These meta tags help link your website to your mobile app.

Verification Tags

<meta name="google-site-verification" content="verification_token">

These are used to verify ownership of your site with various services like Google Search Console, Bing Webmaster Tools, etc.

Best Practices

  • Always include the basic meta tags (charset, viewport, description) in every HTML document.
  • Keep your description meta tag concise (around 150-160 characters) and relevant to the page content.
  • Use Open Graph and Twitter Card meta tags if your content is likely to be shared on social media.
  • Avoid using the refresh meta tag for redirects when possible; use server-side redirects instead.
  • Don't block zooming with user-scalable=no as it creates accessibility issues.
  • Use the canonical link to prevent duplicate content issues.
  • Keep meta tags in the <head> section of your HTML document.
  • Test your meta tags using tools like the Facebook Sharing Debugger, Twitter Card Validator, and Google's Rich Results Test.

Example of a Well-Structured Head Section

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
    <meta name="description" content="A concise description of the page content.">
    <meta name="author" content="Author Name">
    <meta name="robots" content="index, follow">
    
    <!-- Open Graph / Facebook -->
    <meta property="og:type" content="website">
    <meta property="og:url" content="https://example.com/page">
    <meta property="og:title" content="Page Title">
    <meta property="og:description" content="A concise description of the page content.">
    <meta property="og:image" content="https://example.com/image.jpg">
    
    <!-- Twitter -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:title" content="Page Title">
    <meta name="twitter:description" content="A concise description of the page content.">
    <meta name="twitter:image" content="https://example.com/image.jpg">
    
    <link rel="canonical" href="https://example.com/page">
    <link rel="icon" href="/favicon.ico">
    <link rel="stylesheet" href="/styles.css">
</head>
<body>
    <!-- Page content -->
</body>
</html>
