Link Attributes

The target Attribute

The target attribute specifies where to open the linked document:

  • Open in new tab (_blank)
  • Open in same frame (_self - default)
  • Open in parent frame (_parent)
  • Open in full body of window (_top)

The title Attribute

The title attribute provides additional information about the link. Hover over the link to see it:

Example.com with title

The rel Attribute

The rel attribute specifies the relationship between the current document and the linked document:

  • Example.com (nofollow) - Tells search engines not to follow this link
  • Example.com (noopener) - Prevents the new page from accessing the window.opener property
  • Example.com (noreferrer) - Prevents passing the referrer information to the linked website
  • Example.com (noopener noreferrer) - Combination of both

Internal Page Links (Anchor Links)

Internal links (also called anchor links) point to specific sections within the same page:

The links in the Contents section at the top of this page are examples of internal links. They use the id attribute of elements as their targets.

To create an internal link:

  1. Add an id attribute to the element you want to link to: <h2 id="section-name">Section Title</h2>
  2. Create a link with a hash (#) followed by the id: <a href="#section-name">Go to Section</a>

Link States

Links can have different states:

  • Unvisited Link - A link you haven't clicked yet (typically blue)
  • Visited Link - A link you have clicked (typically purple)
  • Active Link - A link at the moment it is clicked

Note: The appearance of these states depends on the browser's default styles or CSS styling.

Best Practices for Links

  • Use descriptive link text that clearly indicates where the link will take the user
  • Avoid generic phrases like "click here" or "read more"
  • Use the title attribute to provide additional information when needed
  • Use target="_blank" with rel="noopener noreferrer" for security when opening links in new tabs
  • Ensure links are accessible by making them distinguishable from regular text
  • Check that all links work correctly and point to the intended destinations
