Basic Form Structure

The basic structure of an HTML form consists of the <form> element, which contains various form controls:

<form action="/submit-form" method="post">
    <!-- Form elements go here -->
    <input type="text" name="username">
    <input type="submit" value="Submit">
</form>

The action attribute specifies where to send the form data when submitted, and the method attribute specifies the HTTP method to use (usually "get" or "post").

Form Attributes

Common form attributes include:

  • action - Specifies where to send the form data when submitted
  • method - Specifies the HTTP method to use (get or post)
  • target - Specifies where to display the response after submitting the form
  • autocomplete - Specifies whether the browser should autocomplete the form
  • novalidate - Specifies that the form should not be validated when submitted
  • enctype - Specifies how the form data should be encoded when submitted (important for file uploads)

Best Practices for Forms

  • Always use label elements for form controls to improve accessibility
  • Group related form elements with fieldset and legend
  • Use appropriate input types for different data (email, tel, number, etc.)
  • Include proper validation with required fields and patterns
  • Provide clear instructions and error messages
  • Use placeholder text to provide hints, not to replace labels
  • Ensure your form is keyboard-accessible
  • Test your form on different devices and browsers
