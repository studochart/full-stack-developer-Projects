HTML Div and Span Elements

The <div> and <span> elements are container elements used to group and style content. They don't have any semantic meaning themselves but are essential for layout and styling. This page demonstrates how to use these elements effectively.

The Div Element

The <div> element is a block-level container used to group larger sections of content or to create layout structures. It's one of the most commonly used HTML elements for layout purposes.

Basic Div

This is a paragraph inside a div.

This is another paragraph in the same div.

HTML code:

<div style="border: 1px solid #ccc; padding: 10px; margin-bottom: 15px;">
    <p>This is a paragraph inside a div.</p>
    <p>This is another paragraph in the same div.</p>
</div>

Div with ID and Class

Information Section

This div has both an ID and a class attribute, which can be used for styling and JavaScript selection.

HTML code:

<div id="info-section" class="content-box" style="background-color: #f0f0f0; padding: 15px; border-radius: 5px; margin-bottom: 15px;">
    <h3>Information Section</h3>
    <p>This div has both an ID and a class attribute, which can be used for styling and JavaScript selection.</p>
</div>

The Span Element

The <span> element is an inline container used to mark up a part of a text or a part of a document. It's useful for applying styles to specific portions of text without breaking the flow.

Basic Span

This paragraph contains red text using a span element.

HTML code:

<p>This paragraph contains <span style="color: red;">red text</span> using a span element.</p>

Span with ID and Class

The highlighted text in this paragraph has both an ID and a class attribute.

HTML code:

<p>The <span id="highlight" class="important" style="background-color: yellow; font-weight: bold;">highlighted text</span> in this paragraph has both an ID and a class attribute.</p>

Differences Between Div and Span

The main differences between <div> and <span> elements are:

Feature Div Span
Display Type Block-level element Inline element
New Line Starts on a new line and takes up the full width available Does not start on a new line and only takes up as much width as necessary
Usage Used for larger sections of content or layout structures Used for small portions of text or inline elements
Content Can contain block-level and inline elements Should only contain inline elements or text

Nesting Divs and Spans

Both <div> and <span> elements can be nested within each other to create complex structures.

Nested Divs

Outer Div

This is content in the outer div.

Inner Div 1

This is content in the first inner div.

Inner Div 2

This is content in the second inner div.

HTML code:

<div style="border: 2px solid black; padding: 15px; margin-bottom: 15px;">
    <h3>Outer Div</h3>
    <p>This is content in the outer div.</p>
    
    <div style="border: 2px dashed blue; padding: 10px; margin: 10px 0;">
        <h4>Inner Div 1</h4>
        <p>This is content in the first inner div.</p>
    </div>
    
    <div style="border: 2px dashed green; padding: 10px;">
        <h4>Inner Div 2</h4>
        <p>This is content in the second inner div.</p>
    </div>
</div>

Nested Spans

This paragraph contains blue text with a bold word and an underlined word.

HTML code:

<p>
    This paragraph contains 
    <span style="color: blue;">
        blue text with a 
        <span style="font-weight: bold;">bold</span> 
        word and an 
        <span style="text-decoration: underline;">underlined</span> 
        word.
    </span>
</p>

Divs with Spans

Product Information

Product Name: Super Gadget 3000
Price: $99.99
Availability: Out of Stock

HTML code:

<div style="background-color: #f0f0f0; padding: 15px; border-radius: 5px; margin-bottom: 15px;">
    <h3>Product Information</h3>
    <p>
        Product Name: <span style="font-weight: bold;">Super Gadget 3000</span><br>
        Price: <span style="color: green;">$99.99</span><br>
        Availability: <span style="color: red;">Out of Stock</span>
    </p>
</div>

Semantic Alternatives

While <div> and <span> are useful generic containers, HTML5 introduced several semantic elements that should be used when appropriate:

Semantic Alternatives to Div

Semantic Element Purpose
<header> Represents introductory content, typically a group of introductory or navigational aids
<nav> Represents a section of a page that contains navigation links
<main> Represents the main content of the document
<article> Represents a self-contained composition in a document
<section> Represents a standalone section of content
<aside> Represents content that is tangentially related to the content around it
<footer> Represents a footer for its nearest sectioning content or sectioning root element

Semantic Alternatives to Span

Semantic Element Purpose
<em> Represents emphasized text
<strong> Represents important text
<mark> Represents text that is marked or highlighted
<cite> Represents the title of a work
<code> Represents computer code
<time> Represents a specific period in time
